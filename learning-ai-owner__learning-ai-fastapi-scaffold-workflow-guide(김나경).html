1. 먼저 이해할 것

이 작업은 AI 기능 구현이 아니라 learning-ai 서비스의 실행 가능한 골격을 만드는 작업이다. Claude API, OpenAI Embedding, pgvector, Kafka Consumer는 이후 Step에서 붙인다. W1 Step 1의 목표는 FastAPI 앱이 켜지고, /health가 응답하며, pytest와 Docker 빌드가 통과하는 상태다.

완료 기준

uvicorn app.main:app --host 0.0.0.0 --port 8090 실행 후 GET /health가 200 OK와 서비스명/버전/상태를 반환하고, pytest 및 Dockerfile 빌드가 통과해야 한다.

범위 제한

골격 단계에서는 DB 연동, Repository, AI 모델 호출, 사용자 인증, Flutter 화면 연동을 구현하지 않는다. 공개 API는 /health만 허용한다.

4. 의존성 기준

분류 패키지 용도 비고
Runtime fastapi API 프레임워크, Pydantic v2 기반 검증, OpenAPI 문서 자동 생성 기술 스택 문서 권장: 0.130+ 또는 최신 0.136.x
Runtime uvicorn[standard] ASGI 서버, 개발 환경 실행 Task 포트: 8090
Runtime pydantic-settings 환경변수 기반 Settings 모델 관리 시크릿은 코드에 하드코딩하지 않는다
Test pytest, pytest-asyncio, httpx FastAPI endpoint 테스트 ASGI 앱 직접 테스트에 사용
Future anthropic, openai Claude, Embedding API 연동 Step 2/3에서 추가한다

5. 진행 순서

Step 1. TASK 시작

1.1
  • Step Goal / Done When / Scope / Input을 TASK_learning-ai에서 확인한다.
  • PRD_W1의 FR-LA-001 Health endpoint 요구사항을 확인한다.
  • Duration 0.5일 기준으로 AI/DB/Kafka 구현을 제외한다.

Step 2. 요구사항 분석

1.2
  • FastAPI + uvicorn 기본 실행 구조를 확인한다.
  • app/, app/api/, app/core/, app/services/ 구조를 설계한다.
  • Python 3.12+ 의존성을 pyproject.toml에 정리한다.
  • Instructions 초안을 TASK 문서 또는 PR 설명에 반영한다.

Step 3. Security 1차 검토

1.3
  • 골격 단계이므로 인증 필요 여부는 No로 기록한다.
  • 권한 종류는 없음으로 기록한다.
  • 공개 API는 /health만 Yes로 기록한다.
  • 결과를 TASK Constraints에 반영한다.

Step 4. ERD 설계

1.4
  • 골격 단계라 DB 연동은 해당 없음으로 처리한다.
  • 프로젝트 디렉토리 구조도를 README나 PR 설명에 작성한다.
  • 실제 작업량을 보고 Duration final을 갱신한다.

Step 5. Security 2차 검토

1.5
  • 민감 정보 암호화는 비해당으로 기록한다.
  • .env.example에는 시크릿 플레이스홀더만 작성한다.
  • pydantic-settings 기반 Settings 모델을 확인한다.
  • 결과를 TASK Constraints에 반영한다.

Step 6. DTO / Entity 설계

1.6
  • HealthResponse Pydantic 모델을 정의한다.
  • Settings 모델을 정의한다.
  • Output Format에 Health 응답 JSON과 Docker 빌드 로그를 반영한다.

Step 7. Repository 구현

1.7
  • 골격 단계라 Repository는 해당 없음으로 처리한다.
  • Poetry 또는 uv 중 하나를 정해 pyproject.toml 기반으로 초기화한다.

Step 8. Service + Test

1.8
  • FastAPI app 인스턴스를 생성한다.
  • /health 엔드포인트를 구현한다.
  • pytest 설정과 health 테스트를 작성한다.
  • 테스트 통과 결과를 남긴다.

Step 9. Controller + Docker

1.9
  • Dockerfile을 작성한다.
  • docker-compose.yml에 learning-ai 서비스를 추가한다.
  • uvicorn이 포트 8090에서 실행되는지 확인한다.

Step 10. View + Test

1.10
  • Flutter 화면 연동은 해당 없음으로 처리한다.
  • Docker 컨테이너 실행 후 /health 응답을 확인한다.

6. 구현 예시

pyproject.toml

[project]
name = "synapse-learning-ai"
version = "0.1.0"
description = "Synapse learning-ai FastAPI service"
requires-python = ">=3.12"
dependencies = [
  "fastapi>=0.130.0",
  "uvicorn[standard]>=0.46.0",
  "pydantic-settings>=2.3.0"
]

[project.optional-dependencies]
dev = [
  "pytest>=9.0.0",
  "pytest-asyncio>=0.24.0",
  "httpx>=0.28.0"
]

[tool.pytest.ini_options]
asyncio_mode = "auto"
testpaths = ["tests"]

app/core/settings.py

from pydantic_settings import BaseSettings, SettingsConfigDict


class Settings(BaseSettings):
    service_name: str = "synapse-learning-ai"
    version: str = "0.1.0"
    environment: str = "local"

    model_config = SettingsConfigDict(
        env_file=".env",
        env_prefix="LEARNING_AI_",
        extra="ignore",
    )


settings = Settings()

app/schemas/health.py

from pydantic import BaseModel


class HealthResponse(BaseModel):
    service: str
    version: str
    status: str

app/api/health.py

from fastapi import APIRouter

from app.core.settings import settings
from app.schemas.health import HealthResponse

router = APIRouter(tags=["health"])


@router.get("/health", response_model=HealthResponse)
async def health() -> HealthResponse:
    return HealthResponse(
        service=settings.service_name,
        version=settings.version,
        status="ok",
    )

app/main.py

from fastapi import FastAPI

from app.api.health import router as health_router
from app.core.settings import settings


app = FastAPI(
    title=settings.service_name,
    version=settings.version,
)

app.include_router(health_router)

tests/test_health.py

from httpx import ASGITransport, AsyncClient

from app.main import app


async def test_health_returns_ok() -> None:
    transport = ASGITransport(app=app)

    async with AsyncClient(
        transport=transport,
        base_url="http://test",
    ) as client:
        response = await client.get("/health")

    assert response.status_code == 200
    assert response.json()["status"] == "ok"

7. Docker와 Compose 예시

Dockerfile

FROM python:3.12-slim AS runtime

ENV PYTHONDONTWRITEBYTECODE=1
ENV PYTHONUNBUFFERED=1

WORKDIR /app

COPY pyproject.toml ./
RUN pip install --no-cache-dir ".[dev]"

COPY app ./app

EXPOSE 8090
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8090"]

docker-compose.yml 추가

services:
  learning-ai:
    build:
      context: ./learning-ai
    container_name: synapse-learning-ai
    ports:
      - "8090:8090"
    env_file:
      - ./learning-ai/.env
    healthcheck:
      test: ["CMD", "python", "-c", "import urllib.request; urllib.request.urlopen('http://localhost:8090/health')"]
      interval: 10s
      timeout: 3s
      retries: 5
운영용 Dockerfile 조정

예시는 골격 확인용이다. 실제 운영 이미지는 dev 의존성 제외, non-root user, lockfile 기반 설치, 취약점 스캔을 추가하는 편이 맞다.
