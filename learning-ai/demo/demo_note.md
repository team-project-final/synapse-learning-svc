# 운영체제 기초 — 프로세스, 스레드, 메모리 관리

## 프로세스와 스레드

**프로세스(Process)**는 실행 중인 프로그램의 인스턴스다. 운영체제는 각 프로세스에 독립된 주소 공간(코드, 데이터, 힙, 스택), 파일 핸들, PCB(Process Control Block)를 할당한다. 프로세스 간 통신(IPC)은 파이프, 소켓, 공유 메모리 등을 통해 이루어지며 오버헤드가 크다.

**스레드(Thread)**는 프로세스 내의 실행 단위다. 같은 프로세스의 스레드들은 코드·데이터·힙을 공유하고, 스택과 레지스터만 독립적으로 가진다. 공유 메모리를 사용하기 때문에 컨텍스트 스위칭 비용이 프로세스보다 훨씬 작다. 단, 공유 자원에 대한 동기화(뮤텍스, 세마포어)가 필요하며 하나의 스레드 오류가 전체 프로세스에 영향을 줄 수 있다.

| 항목 | 프로세스 | 스레드 |
|------|----------|--------|
| 주소 공간 | 독립 | 공유 (스택만 독립) |
| 생성 비용 | 높음 | 낮음 |
| 통신 방식 | IPC | 공유 메모리 |
| 오류 영향 범위 | 해당 프로세스만 | 전체 프로세스 |

## CPU 스케줄링

운영체제는 여러 프로세스/스레드가 CPU를 효율적으로 사용하도록 스케줄링 알고리즘을 적용한다.

- **FCFS(First Come First Served)**: 먼저 온 작업부터 처리. 구현이 단순하지만 긴 작업이 짧은 작업을 막는 Convoy Effect 발생.
- **SJF(Shortest Job First)**: 실행 시간이 짧은 작업을 우선 처리. 평균 대기 시간이 최소지만 실행 시간 예측이 어렵고 기아(Starvation) 문제 발생.
- **Round Robin**: 각 프로세스에 동일한 시간 할당량(Time Quantum)을 부여하고 순환 처리. 공정하지만 Time Quantum 설정이 성능에 큰 영향을 미침.
- **Priority Scheduling**: 우선순위가 높은 작업을 먼저 처리. 낮은 우선순위 프로세스의 기아 문제를 Aging으로 해결.

**컨텍스트 스위칭(Context Switching)**은 CPU가 현재 프로세스의 상태를 PCB에 저장하고 다음 프로세스의 상태를 복원하는 과정이다. 이 시간 동안 CPU는 유용한 작업을 수행하지 못하므로 오버헤드로 간주된다.

## 메모리 관리 — 가상 메모리와 페이징

**가상 메모리(Virtual Memory)**는 물리 메모리 크기를 초과하는 프로그램을 실행할 수 있게 해주는 기법이다. 각 프로세스는 연속된 가상 주소 공간을 가지는 것처럼 동작하며, 실제 물리 메모리와의 매핑은 MMU(Memory Management Unit)가 담당한다.

**페이징(Paging)**은 가상 주소 공간을 고정 크기의 **페이지(Page)**, 물리 메모리를 같은 크기의 **프레임(Frame)**으로 나누는 방식이다. **페이지 테이블**이 가상 페이지 번호(VPN)를 물리 프레임 번호(PFN)로 변환한다. 페이지 테이블 접근 오버헤드를 줄이기 위해 **TLB(Translation Lookaside Buffer)**라는 캐시를 사용한다.

**페이지 폴트(Page Fault)**는 프로세스가 접근한 페이지가 물리 메모리에 없을 때 발생한다. 운영체제는 디스크의 스왑 영역에서 해당 페이지를 메모리로 로드한다. 메모리가 가득 찼을 때 교체할 페이지를 선택하는 알고리즘(LRU, FIFO, OPT)을 **페이지 교체 알고리즘**이라 한다.

## 교착 상태 (Deadlock)

교착 상태는 두 개 이상의 프로세스가 서로 상대방이 점유한 자원을 기다리며 무한히 블로킹되는 상황이다. 발생 조건은 네 가지가 동시에 성립해야 한다.

1. **상호 배제(Mutual Exclusion)**: 자원을 한 번에 하나의 프로세스만 사용
2. **점유 대기(Hold and Wait)**: 자원을 점유한 채 다른 자원 대기
3. **비선점(No Preemption)**: 다른 프로세스의 자원을 강제로 빼앗을 수 없음
4. **순환 대기(Circular Wait)**: 프로세스들이 원형으로 자원을 대기

예방, 회피(은행원 알고리즘), 탐지 및 회복으로 처리한다.
